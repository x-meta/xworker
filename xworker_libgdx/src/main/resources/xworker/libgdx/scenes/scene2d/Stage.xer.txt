#$@text#$@
<p><span style="background-color:rgb(255, 255, 255); color:rgb(71, 71, 71)">A 2D scene graph containing hierarchies of </span><a href="http://libgdx.badlogicgames.com/nightlies/docs/api/com/badlogic/gdx/scenes/scene2d/Actor.html" style="font: 14px/19.6px &quot;DejaVu Serif&quot;, Georgia, &quot;Times New Roman&quot;, Times, serif; text-align: left; color: blue; text-transform: none; text-indent: 0px; letter-spacing: normal; text-decoration: none; word-spacing: 0px; white-space: normal; widows: 1; font-size-adjust: none; font-stretch: normal; background-color: rgb(255, 255, 255); -webkit-text-stroke-width: 0px;" title="class in com.badlogic.gdx.scenes.scene2d"><code>actors</code></a><span style="background-color:rgb(255, 255, 255); color:rgb(71, 71, 71)">. Stage handles the viewport and distributes input events.</span></p>

<p style="text-align: left;"><a href="http://libgdx.badlogicgames.com/nightlies/docs/api/com/badlogic/gdx/scenes/scene2d/Stage.html#setViewport-com.badlogic.gdx.utils.viewport.Viewport-" style="color: blue; text-decoration: none;"><code>setViewport(Viewport)</code></a> controls the coordinates used within the stage and sets up the camera used to convert between stage coordinates and screen coordinates.</p>

<p style="text-align: left;">A stage must receive input events so it can distribute them to actors. This is typically done by passing the stage to <a href="http://libgdx.badlogicgames.com/nightlies/docs/api/com/badlogic/gdx/Input.html#setInputProcessor-com.badlogic.gdx.InputProcessor-" style="color: blue; text-decoration: none;"><code>Gdx.input.setInputProcessor</code></a>. An <a href="http://libgdx.badlogicgames.com/nightlies/docs/api/com/badlogic/gdx/InputMultiplexer.html" style="color: blue; text-decoration: none;" title="class in com.badlogic.gdx"><code>InputMultiplexer</code></a> may be used to handle input events before or after the stage does. If an actor handles an event by returning true from the input method, then the stage&#39;s input method will also return true, causing subsequent InputProcessors to not receive the event.</p>

<p style="text-align: left;">The Stage and its constituents (like Actors and Listeners) are not thread-safe and should only be updated and queried from a single thread (presumably the main render thread). Methods should be reentrant, so you can update Actors and Stages from within callbacks and handlers.</p>
