#$@text#$@
<p style="text-align: left;">A Pixmap represents an image in memory. It has a width and height expressed in pixels as well as a <a href="http://libgdx.badlogicgames.com/nightlies/docs/api/com/badlogic/gdx/graphics/Pixmap.Format.html" style="color: blue; text-decoration: none;" title="enum in com.badlogic.gdx.graphics"><code>Pixmap.Format</code></a> specifying the number and order of color components per pixel. Coordinates of pixels are specified with respect to the top left corner of the image, with the x-axis pointing to the right and the y-axis pointing downwards.</p>

<p style="text-align: left;">By default all methods use blending. You can disable blending with <a href="http://libgdx.badlogicgames.com/nightlies/docs/api/com/badlogic/gdx/graphics/Pixmap.html#setBlending-com.badlogic.gdx.graphics.Pixmap.Blending-" style="color: blue; text-decoration: none;"><code>setBlending(Blending)</code></a>. The <a href="http://libgdx.badlogicgames.com/nightlies/docs/api/com/badlogic/gdx/graphics/Pixmap.html#drawPixmap-com.badlogic.gdx.graphics.Pixmap-int-int-int-int-int-int-int-int-" style="color: blue; text-decoration: none;"><code>drawPixmap(Pixmap, int, int, int, int, int, int, int, int)</code></a> method will scale and stretch the source image to a target image. There either nearest neighbour or bilinear filtering can be used.</p>

<p style="text-align: left;">A Pixmap stores its data in native heap memory. It is mandatory to call <a href="http://libgdx.badlogicgames.com/nightlies/docs/api/com/badlogic/gdx/graphics/Pixmap.html#dispose--" style="color: blue; text-decoration: none;"><code>dispose()</code></a> when the pixmap is no longer needed, otherwise memory leaks will result</p>
