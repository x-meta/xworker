<p style="widows: 2; text-transform: none; text-indent: 0px; font: medium Simsun; white-space: normal; orphans: 2; letter-spacing: normal; color: rgb(0,0,0); word-spacing: 0px; -webkit-text-size-adjust: auto; -webkit-text-stroke-width: 0px">Sets a<span class="Apple-converted-space">&nbsp;</span><a href="http://ant.apache.org/manual/using.html#properties">property</a><span class="Apple-converted-space">&nbsp;</span>(by name and value), or set of properties (from file or resource) in the project. Properties are case sensitive.</p>
<p><span style="widows: 2; text-transform: none; text-indent: 0px; display: inline !important; font: medium Simsun; white-space: normal; orphans: 2; float: none; letter-spacing: normal; color: rgb(0,0,0); word-spacing: 0px; -webkit-text-size-adjust: auto; -webkit-text-stroke-width: 0px">Properties are immutable: whoever sets a property first freezes it for the rest of the build; they are most definitely not variables.</span></p>
<p style="widows: 2; text-transform: none; text-indent: 0px; font: medium Simsun; white-space: normal; orphans: 2; letter-spacing: normal; color: rgb(0,0,0); word-spacing: 0px; -webkit-text-size-adjust: auto; -webkit-text-stroke-width: 0px">There are seven ways to set properties:</p>
<ul style="widows: 2; text-transform: none; text-indent: 0px; font: medium Simsun; white-space: normal; orphans: 2; letter-spacing: normal; color: rgb(0,0,0); word-spacing: 0px; -webkit-text-size-adjust: auto; -webkit-text-stroke-width: 0px">
    <li>By supplying both the<span class="Apple-converted-space">&nbsp;</span><i>name</i><span class="Apple-converted-space">&nbsp;</span>and one of<span class="Apple-converted-space">&nbsp;</span><i>value</i><span class="Apple-converted-space">&nbsp;</span>or<span class="Apple-converted-space">&nbsp;</span><i>location</i><span class="Apple-converted-space">&nbsp;</span>attribute.</li>
    <li>By supplying the<span class="Apple-converted-space">&nbsp;</span><i>name</i><span class="Apple-converted-space">&nbsp;</span>and nested text.</li>
    <li>By supplying both the<span class="Apple-converted-space">&nbsp;</span><i>name</i><span class="Apple-converted-space">&nbsp;</span>and<span class="Apple-converted-space">&nbsp;</span><i>refid</i><span class="Apple-converted-space">&nbsp;</span>attribute.</li>
    <li>By setting the<span class="Apple-converted-space">&nbsp;</span><i>file</i><span class="Apple-converted-space">&nbsp;</span>attribute with the filename of the property file to load. This property file has the format as defined by the file used in the class java.util.Properties, with the same rules about how non-ISO8859-1 characters must be escaped.</li>
    <li>By setting the<span class="Apple-converted-space">&nbsp;</span><i>url</i><span class="Apple-converted-space">&nbsp;</span>attribute with the url from which to load the properties. This url must be directed to a file that has the format as defined by the file used in the class java.util.Properties.</li>
    <li>By setting the<span class="Apple-converted-space">&nbsp;</span><i>resource</i><span class="Apple-converted-space">&nbsp;</span>attribute with the resource name of the property file to load. A resource is a property file on the current classpath, or on the specified classpath.</li>
    <li>By setting the<span class="Apple-converted-space">&nbsp;</span><i>environment</i><span class="Apple-converted-space">&nbsp;</span>attribute with a prefix to use. Properties will be defined for every environment variable by prefixing the supplied name and a period to the name of the variable.</li>
</ul>
<p style="widows: 2; text-transform: none; text-indent: 0px; font: medium Simsun; white-space: normal; orphans: 2; letter-spacing: normal; color: rgb(0,0,0); word-spacing: 0px; -webkit-text-size-adjust: auto; -webkit-text-stroke-width: 0px">Although combinations of these ways are possible, only one should be used at a time. Problems might occur with the order in which properties are set, for instance.</p>
<p style="widows: 2; text-transform: none; text-indent: 0px; font: medium Simsun; white-space: normal; orphans: 2; letter-spacing: normal; color: rgb(0,0,0); word-spacing: 0px; -webkit-text-size-adjust: auto; -webkit-text-stroke-width: 0px">The value part of the properties being set, might contain references to other properties. These references are resolved at the time these properties are set. This also holds for properties loaded from a property file.</p>
<p style="widows: 2; text-transform: none; text-indent: 0px; font: medium Simsun; white-space: normal; orphans: 2; letter-spacing: normal; color: rgb(0,0,0); word-spacing: 0px; -webkit-text-size-adjust: auto; -webkit-text-stroke-width: 0px">A list of predefined properties can be found<span class="Apple-converted-space">&nbsp;</span><a href="http://ant.apache.org/manual/properties.html#built-in-props">here</a>.</p>
<p style="widows: 2; text-transform: none; text-indent: 0px; font: medium Simsun; white-space: normal; orphans: 2; letter-spacing: normal; color: rgb(0,0,0); word-spacing: 0px; -webkit-text-size-adjust: auto; -webkit-text-stroke-width: 0px">Since Apache Ant 1.7.1 it is possible to load properties defined in xml according to<span class="Apple-converted-space">&nbsp;</span><a href="http://java.sun.com/dtd/properties.dtd">Suns DTD</a>, if Java5+ is present. For this the name of the file, resource or url has to end with<span class="Apple-converted-space">&nbsp;</span><tt>.xml</tt>.</p>
