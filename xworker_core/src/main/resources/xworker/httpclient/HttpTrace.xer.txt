<p>HTTP TRACE method.</p>

<p>The HTTP TRACE method is defined in section 9.6 of <a href="http://www.ietf.org/rfc/rfc2616.txt">RFC2616</a>:</p>

<blockquote>The TRACE method is used to invoke a remote, application-layer loop- back of the request message. The final recipient of the request SHOULD reflect the message received back to the client as the entity-body of a 200 (OK) response. The final recipient is either the origin server or the first proxy or gateway to receive a Max-Forwards value of zero (0) in the request (see section 14.31). A TRACE request MUST NOT include an entity.</blockquote>

<p><strong>可以重写的方法：</strong></p>

<ul>
	<li><strong>getHeaders</strong><br />
	返回要设置的Headers。<br />
	&nbsp;</li>
	<li><strong>getHttpParams</strong><br />
	返回要设置的HttpParams。<br />
	&nbsp;</li>
	<li><strong>getHttpClient</strong><br />
	返回HttpClient。<br />
	&nbsp;</li>
	<li><strong>getUri</strong><br />
	返回要请求的uri。</li>
</ul>

<p><strong>事件：</strong></p>

<ul>
	<li><strong>onSuccess</strong><br />
	当httpGet请求成功后触发，参数httpClient，request,response,entity，具体类型见Apache的HttpClient包。<br />
	默认返回请求返回的字符串。<br />
	&nbsp;</li>
	<li><strong>onException</strong><br />
	当HttpGet请求发生Java异常时调用，参数httpClient，request, exception.<br />
	默认返回exception。</li>
</ul>

<p><strong>返回值: </strong>返回onSuccess或onException的返回值，其中如果onException返回的是Exception对象，那么抛出异常。</p>
