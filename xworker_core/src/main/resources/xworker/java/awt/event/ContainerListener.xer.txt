<p><span style="background-color:rgb(255, 255, 255); color:rgb(53, 56, 51)">The listener interface for receiving container events. The class that is interested in processing a container event either implements this interface (and all the methods it contains) or extends the abstract </span><code>ContainerAdapter</code><span style="background-color:rgb(255, 255, 255); color:rgb(53, 56, 51)"> class (overriding only the methods of interest). The listener object created from that class is then registered with a component using the component&#39;s </span><code>addContainerListener</code><span style="background-color:rgb(255, 255, 255); color:rgb(53, 56, 51)"> method. When the container&#39;s contents change because a component has been added or removed, the relevant method in the listener object is invoked, and the </span><code>ContainerEvent</code><span style="background-color:rgb(255, 255, 255); color:rgb(53, 56, 51)"> is passed to it.</span></p>

<p style="text-align: left;">Container events are provided for notification purposes ONLY; The AWT will automatically handle add and remove operations internally so the program works properly regardless of whether the program registers a <code>ComponentListener</code> or not.</p>
