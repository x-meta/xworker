<p><strong>JavaAction</strong></p>

<p>&nbsp;&nbsp;&nbsp; JavaAction的作用是执行Java代码，JavaAction是基本的动作，其它类型的动作模型最后也是靠JavaAction来解释执行。</p>

<p><strong>调用Java的方法</strong></p>

<p>&nbsp;&nbsp;&nbsp; 是通过调用Java的方法来实现的。</p>

<ul>
	<li><strong>固定参数的Java方法</strong><br />
	如: public static String run(ActionContext actionContext)，参数固定为ActionContext，返回值类型和是否是静态不限。<br />
	&nbsp;</li>
	<li><strong>多个参数的Java方法</strong><br />
	由于通过发射有时不能正确获取参数的名字，而参数名是从变量上下文取变量的，所以需要通过注解来说明，如下面的示例：<br />
	<strong>@ActionParams(names=&quot;p1,p2,p3&quot;)</strong><br />
	public static String run(Object p1,&nbsp;String p2,&nbsp; int p3, ActionContext actionContext)<br />
	最后一个参数如果是ActionContext，那么可以不用再注解里说明。<br />
	&nbsp;</li>
	<li><strong>非静态方法</strong><br />
	非静态方法执行时必须要有一个所属的对象，因此需要下面的注解。<br />
	<strong>@ActionClass(creator=&quot;xxx&quot;)</strong>，可选，针对类的，creator用于指明一个实例化对象的方法，实例化对象的方法必须是静态的且参数固定为ActionCotnext。<br />
	<strong>@ActionField(name=&quot;xxx&quot;)</strong>，针对属性，标记在执行被调用的方法前，从变量上下文中取xxx变量为其赋值。</li>
</ul>

<p><strong>外部Java</strong></p>

<p>&nbsp;&nbsp;&nbsp; 是指在Java项目中或者已经打包到类库中的Java类，常用该方法。</p>

<p><strong>内部Java</strong></p>

<p>&nbsp;&nbsp;&nbsp; 是指在当前事物模型的管理器下编写的Java代码。很少使用，很久没测是否功能还正常。</p>

<p><strong>Java代码</strong></p>

<p>&nbsp;&nbsp;&nbsp; 直接在当前模型里编写Java代码。很少使用，很久没测是否功能还正常。</p>
