<p><strong>会话</strong></p>

<p>会话是交互式编程的一种实现，可以通过会话编程或使用已有的应用。</p>

<p><strong>使用会话的方法</strong></p>

<ol>
	<li>在会话树中选择一个会话，此时对话树中会显示可选的对话。</li>
	<li>在对话树中选择一个对话，在对话面板显示相关操作界面，然后根据操作界面的内容实现交互。</li>
</ol>

<p><strong>&nbsp;&nbsp;&nbsp; 注：</strong>会话树在会话Tab里，对话树在会话树的正下方。</p>

<p><strong>创建会话的方法</strong></p>

<ol>
	<li>把任意一个事物发送到标识为xworker_session_swt_sessionTree的UIHandler上，就可以在会话树中显示该事物。</li>
</ol>

<p><strong>创建对话的方法</strong></p>

<ol>
	<li>在会话树中选中一个会话时会先清空对话树。</li>
	<li>然后会执行当前会话的run方法。</li>
	<li>在会话的run方法把对话事物发送到标识为xworker_session_swt_sessionDialogTree的UIHandler上，就可以在对话树中显示对话了。</li>
	<li>除了在会话树中选择会话可以创建对话，也可以在任何时候把对话事物发送到标识为xworker_session_swt_sessionDialogTree的UIHandler上。</li>
</ol>

<p><strong>如何创建默认会话（系统启动就有的会话）</strong></p>

<ol>
	<li>系统启动时会寻找每个事物管理器下的&lt;c1&gt;.&lt;c2&gt;.config.Session事物，其中c1和c2为目录名。</li>
	<li>如果Session事物存在，那么执行Session事物的run方法，在run方法里可以注册会话到会话树。</li>
</ol>

<p><strong>会话的基本结构</strong></p>

<p>经过仔细研究，XWorker认为基本的会话包含两个概念：会话、对话。</p>

<p><strong>会话</strong></p>

<p>会话是有一个或者多个对话组成的，目的是共同完成一个任务，或者是围绕一个主题的对话。</p>

<p>同时同一个会话通常有相同的语境，即有同一个全局变量范围。</p>

<p><strong>对话</strong></p>

<p>对话通常是一次交互，比如客户端给出一个界面用户输入数据就是一个对话。</p>
