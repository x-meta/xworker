<h2>Notes</h2>

<div class="section">
<ul>
	<li>The height should be at least 0.15 times the barcode&#39;s length or 0.25 inches.</li>
	<li>The module width (width of narrow element) should be at least 7.5 mils (a mil is 1/1000 inch) or 0.19mm.</li>
	<li>The multiple for the wide element should be between 2.0 and 3.0 <strong>if</strong> the narrow element is greater than 20 mils. If the narrow element is less than 20 mils, the multiple can only range between 2.0 and 2.2.</li>
	<li>Quiet zones must be at least 10 times the module width or 0.10 inches, whichever is larger.</li>
	<li>The intercharacter gap width (igw) is 5.3 times the module width (mw) if mw is less than 10 mils. If mw is 10 mils or greater, the value for igw is 3mw or 53 mils, whichever is greater. However, for quality printers, igw often equals mw.</li>
</ul>
</div>

<p><a name="N1007E"></a><a name="Message+format"></a></p>

<h2>Message format</h2>

<div class="section">
<ul>
	<li>Valid characters are 0-9, A-Z and the characters: - . $ / + % SPACE</li>
	<li>To enable support for all US-ASCII (7-bit) characters, set &quot;extended-charset&quot; to &quot;true&quot;.</li>
	<li>The character &#39;*&#39; is only used as start and stop character and may not be used in the message (unless the extended character set is enabled). The &#39;*&#39; character is automatically added before and after the message even though you may not see it in the human-readable part. (See also &quot;display-start-stop&quot; and &quot;display-checksum&quot; values)</li>
	<li>display-start-stop controls whether the start and stop characters should be included in the human-readable part. Setting display-start-stop to true, automatically sets display-checksum to true, too. Note: this option is ignored if the extended character set is enabled!</li>
	<li>display-checksum controls whether the calculated checksum should be included in the human-readable part. Note: this option is ignored if the extended character set is enabled!</li>
</ul>
</div>
