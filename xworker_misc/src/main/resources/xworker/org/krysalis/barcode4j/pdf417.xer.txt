<h2>Notes</h2>

<div class="section">
<ul>
	<li>This symbology has no human-readable part!</li>
	<li>&quot;ec-level&quot; is the error correction level and must be between 0 and 8.</li>
	<li>If &quot;vertical-quiet-zone&quot; is not set explicitely, it is the same as the horizontal quiet zone.</li>
	<li>Depending on the size of the message, &quot;columns&quot; and &quot;ec-level&quot; may need to be adjusted. Read possible error messages carefully.</li>
	<li>Alternatively, you can remove the &quot;columns&quot; setting and work with &quot;min/max-columns&quot; and &quot;min/max-rows&quot; which allows the symbol to grow and shrink depending on the message. Using the &quot;width-to-height-ratio&quot; you can control the shape of the symbol. (Note: the valid number of rows must be between 3 and 90, the valid number of columns between 1 and 30)</li>
	<li>Using the &quot;columns&quot; setting overrides the &quot;min/max-columns&quot; setting! Or in other words: Setting &quot;columns&quot; sets &quot;min-columns&quot; and &quot;max-columns&quot; to the same value.</li>
	<li>Macro PDF417 functionality is not implemented, yet.</li>
	<li>Compact PDF417 functionality is not implemented, yet.</li>
</ul>
</div>

<p><a name="N10080"></a><a name="Message+format"></a></p>

<h2>Message format</h2>

<div class="section">
<ul>
	<li>All printable ASCII characters are valid characters.</li>
	<li>Byte compaction mode permits all 256 possible 8-bit byte values to be encoded.</li>
	<li>Using only numeric characters allows for smaller symbol sizes.</li>
	<li>Currently, no ECI functionality is available. All characters are interpreted in &quot;cp437&quot; (PC437) encoding.</li>
</ul>
</div>
