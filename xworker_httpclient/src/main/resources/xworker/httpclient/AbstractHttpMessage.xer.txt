<p><strong>设置URI的规则</strong></p>

<p>&nbsp;&nbsp;&nbsp; 使用URIBuilder来构建URI。</p>

<ul>
	<li>使用uri的结果来初始化URIBuilder，支持String和URI两种返回值。</li>
	<li>如果uriPath、uriFragment、uriCharst和uriParams存在则设置。</li>
</ul>

<p><strong>Headers的设置</strong></p>

<p>&nbsp;&nbsp;&nbsp; 可以使用headers属性或Header子节点，二者都可以生效。</p>

<p><strong>RequestConfig</strong></p>

<p>&nbsp;&nbsp;&nbsp; 通过requestConfig属性获取对应对象，或通过requestConfigThing和RequestConfig子节点设置对应的模型。</p>

<p><strong>处理请求的结果</strong></p>

<ul>
	<li><strong>OnSuccess子节点</strong><br />
	可以添加动作，请求无异常抛出时执行。可以使用的变量有httpClient、request、response、entity等。<br />
	&nbsp;</li>
	<li><strong>OnException子节点</strong><br />
	可以添加动作，请求有异常抛出时执行。可以使用的变量有httpClient、request和exception等。<br />
	如果没有设置OnException子节点，发生异常时会抛出。</li>
</ul>

<p><strong>可以重写的方法</strong></p>

<ul>
	<li><strong>void initRequest(HttpRequest request)</strong><br />
	执行前会调用这个方法，可以用来编写额外的初始化request的操作。&nbsp;</li>
</ul>

<p>&nbsp;</p>

#$@text#$@
<p>Represents an HTTP header field.</p>

<p>The HTTP header fields follow the same generic format as that given in Section 3.1 of RFC 822. Each header field consists of a name followed by a colon (&quot;:&quot;) and the field value. Field names are case-insensitive. The field value MAY be preceded by any amount of LWS, though a single SP is preferred.</p>

<pre>
     message-header = field-name &quot;:&quot; [ field-value ]
     field-name     = token
     field-value    = *( field-content | LWS )
     field-content  = &lt;the OCTETs making up the field-value
                      and consisting of either *TEXT or combinations
                      of token, separators, and quoted-string&gt;</pre>

<ul>
	<li><strong>getName</strong><br />
	返回Header的名称。<br />
	&nbsp;</li>
	<li><strong>getValue</strong><br />
	返回Header的值。</li>
</ul>
